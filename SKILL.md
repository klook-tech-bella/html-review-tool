# inject-review — HTML Demo 评审工具注入器

## 用途

给任意 HTML demo 注入一套轻量评审工具条。评审人双击打开 HTML 就能：
- 点页面任意元素留评论（必改 / 建议 / 疑问）
- 评论暂存在浏览器内，可全部导出为一个新的 HTML 文件
- 文件回传给作者，作者双击打开就能看到所有评论标注

**核心特性**：零服务器、零账号、零付费、评论存在 HTML 文件本身。

## 何时触发

- 用户直接喊：`/inject-review`、"注入评审"、"inject review"
- 用户刚生成完一个 HTML demo，说"发给同事评审" / "收集评论" / "让别人 review"
- 用户收到别人发来的 HTML，说"帮我改成可评论的" / "加个评论功能"
- 用户提到"评审 demo"、"review HTML"、"收集反馈"相关需求

## 使用方式

### 基本用法

```bash
python3 inject_comments.py <输入HTML路径> [输出路径]
```

不指定输出路径时，默认生成同目录下的 `<原文件名>_review.html`。

### 典型工作流

1. **作者侧** — 生成完 HTML demo 后：
   ```bash
   python3 inject_comments.py ./my_demo.html
   # → 生成 my_demo_review.html
   ```
   把 `my_demo_review.html` 通过微信 / 飞书 / 邮件发给评审人。

2. **评审人侧** — 收到 HTML 文件：
   - 双击打开（任何浏览器、离线可用）
   - 右上角工具条：填名字 → 点"开启评审" → 点元素留评论
   - 评审完点 "💾 保存" → 下载 `<原名>_review_<评审人>.html`
   - 把这个文件回传给作者

3. **作者侧** — 收到评审后的 HTML：
   - 双击打开 → 自动看到所有评论标注（右上红色数字）
   - 点数字查看具体评论，点 💬 看评论列表
   - 可以继续评审或交给下一位评审人

### 同一份 demo 多人评审

- **串行**：A 评完 → 回传 → B 基于 A 的评审版再评 → 回传（评论累积）
- **并行**：原始版本同时发给 A 和 B → 各自评完 → 作者收到两份 → 手动 / 脚本合并

## 执行步骤（Claude 的行为）

当用户触发这个 skill 时：

1. **确认输入文件**：
   - 如果用户指定了 HTML 路径，直接用
   - 如果用户说"刚才那个 demo" / 没指定，用 Glob 找最近修改的 `*.html`，跟用户确认
   - 如果找不到，问用户路径

2. **运行注入**：
   ```bash
   python3 <skill目录>/inject_comments.py <路径>
   ```

3. **自动打开浏览器预览**（可选，用户没反对就默认打开）：
   ```bash
   open <输出路径>
   ```

4. **告诉用户**：
   - 输出文件路径
   - 简短的评审人使用说明
   - 可以直接把这个文件发给评审人

## 注入的工具条功能

评审人打开 HTML 后看到的 UI：

- **右上角深紫工具条**：名字输入框 / `开启评审` 按钮 / `💬 N` 评论计数 / `💾 保存`
- **评审模式下**：鼠标悬停元素红色虚线高亮；点击元素弹出评论对话框（文本 + 必改/建议/疑问 三选一）
- **已有评论**：被评论元素右上角红色数字标记，点标记看评论，点列表项定位高亮

## 幂等性

脚本自带幂等处理：对已经注入过的 HTML 再次运行，会自动剥离旧注入再重新注入，不会重复。

## 数据安全

- 评论只存在 HTML 文件本身（`<script id="rv-data">` JSON 块）
- 不上传任何服务器
- localStorage 仅用于记住评审人名字
- 文件传输完全由用户掌控

## 限制

- HTML 必须包含 `</body>` 标签（99% 都有）
- 元素定位基于 CSS nth-child 选择器，如果原 HTML 的 DOM 结构被二次编辑，可能有少量标记错位

## 典型问答

**Q: 评审人需要装什么？**
A: 什么都不用。任何浏览器双击 HTML 就能用。

**Q: 需要联网吗？**
A: 不需要。完全离线。

**Q: 评论会不会丢？**
A: 评论存在 HTML 文件本身。评审人点 "💾 保存" 下载新文件后评论就固化了。如果没点保存就关了浏览器，会丢。

**Q: 评审人也要装这个 skill 吗？**
A: 不用。只有"生成 demo 并发起评审"的那方需要。评审人只要有浏览器。
